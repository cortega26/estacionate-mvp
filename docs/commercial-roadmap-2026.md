# Commercial Roadmap 2026

**Date:** 2026-04-26  
**Scope:** Product improvement, conversion improvement, solo-founder execution, pitch readiness

## Objective

Improve Estacionate's chances of converting prospects into pilots and pilots into paid customers without destabilizing the core product.

## Strategic Priorities

1. Improve perceived professionalism.
2. Improve demo effectiveness.
3. Strengthen trust and operational credibility.
4. Reduce friction from first meeting to pilot.
5. Make technical improvements only where they support conversion and delivery.

## Days 1-30: Make the Product Sellable

**Objective:** Improve first impression, demo reliability, and commercial clarity.

### Outcomes

1. Clear positioning by stakeholder.
2. Stable demo environment.
3. Strong trust assets.
4. Cleaner critical UX paths.

### Product Workstream

1. Fix the top five visible UX issues in booking, admin, and guard flows.
2. Standardize labels, statuses, and user-facing error messages.
3. Improve empty states and next-step guidance.
4. Verify mobile usability for resident and guard flows.

### Sales Workstream

1. Finalize ICP and buyer profiles.
2. Write one core positioning statement and three stakeholder-specific value propositions.
3. Define the commercial narrative: pain, cost of inaction, solution, expected outcome.
4. Define a simple pilot offer and initial pricing logic.

### Demo Readiness Workstream

1. Create a demo tenant with realistic data.
2. Prepare a scripted 15-minute demo.
3. Define the demo order: resident, concierge, admin.
4. Eliminate broken states or unclear transitions in the demo flow.

### Trust Workstream

1. Produce a security and reliability one-pager.
2. Document onboarding for a new building.
3. Prepare a short objections FAQ.
4. Capture screenshots and workflow visuals.

### Success Criteria by Day 30

1. The same demo can be run consistently in under 15 minutes.
2. Prospects can understand the value in under 2 minutes.
3. Public materials can explain the product without a live call.
4. Critical demo flows are stable.

## Days 31-60: Improve Conversion Infrastructure

**Objective:** Convert interest into qualified follow-up and pilots.

### Outcomes

1. Stronger acquisition layer.
2. Better commercial packaging.
3. Stronger trust and ROI framing.
4. Lower friction from demo to pilot.

### Marketing and Acquisition Workstream

1. Publish or improve public pages: Home, For Administrators, For Residents, For Staff, Pricing, Security, Book a Demo.
2. If a new framework is used here, prefer Next.js for the public acquisition layer only.
3. Add lead capture and analytics.
4. Create clear CTA paths for demo requests and pilot interest.

### Commercial System Workstream

1. Create a proposal template.
2. Create a pilot template with scope, duration, and success criteria.
3. Create an objection-handling sheet.
4. Build a simple ROI calculator or benefit table.

### Product Improvements Tied to Conversion

1. Make the admin dashboard more executive-friendly.
2. Improve clarity of booking, payment, and validation statuses.
3. Add summary reporting for occupancy and usage.
4. Strengthen traceability in flows prospects care about.

### Delivery Discipline Workstream

1. Standardize frontend API usage.
2. Refactor the most overloaded backend flow.
3. Strengthen payment and webhook safety and logging.
4. Expand checks for critical user flows.

### Success Criteria by Day 60

1. A repeatable funnel exists from contact to demo to pilot.
2. A proposal can be sent within 24 hours of a call.
3. The product demonstrates operational credibility, not just features.
4. A pilot package is ready to offer.

## Days 61-90: Close Deals and Learn Fast

**Objective:** Turn structured interest into pilots and pilots into paid customers.

### Outcomes

1. Pilot execution readiness.
2. Objection-driven refinement.
3. Stronger proof assets.
4. Better prioritization for the next build cycle.

### Pilot Readiness Workstream

1. Create an implementation checklist for a new building.
2. Prepare onboarding by role.
3. Define pilot support and escalation rules.
4. Define first-30-day pilot success metrics.

### Sales Execution Workstream

1. Run consistent outreach to qualified accounts.
2. Track demo-to-follow-up, follow-up-to-pilot, and pilot-to-paid rates.
3. Adjust messaging based on objections.
4. Prioritize prospects with clear operational pain and urgency.

### Proof Generation Workstream

1. Capture screenshots and metrics from pilots.
2. Build one mini case study.
3. Document time saved, process clarity, or reduced complaints.
4. Convert those outcomes into reusable collateral.

### Technical Stabilization Workstream

1. Fix demo or pilot friction immediately.
2. Improve observability in auth, bookings, payments, and validation.
3. Simplify support-heavy workflows.
4. Reassess whether a selective migration is now justified.

### Success Criteria by Day 90

1. A repeatable pilot model is in place.
2. Objections and conversion drop-offs are measured.
3. At least one proof asset is strong enough to support the next sale.
4. The next bottleneck is known: product, trust, pricing, or distribution.
